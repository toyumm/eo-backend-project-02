## 작업 회고
- 리포지토리 인터페이스 정의에는 구현체인 PageRequest를 사용했으나, 서비스에서는 인터페이스인 Pageable을 사용하여 타입이 충돌함
- 리포지토리의 파라미터를 Pagealbe로 수정하여 해결완료

- 테스트 코드에서 @BeforeEach 주해를 사용해서 유저 객체를 생성하였는데 name,email 을 적지 않아서 ConstraintViolationException 발생하였음
- UserEntity에서 컴럼의 벨리데이션 제약조건을 확인하여 빠진 name 과 email을 채워서 확인함

- 강사님의 코드 작성속도와 바로바로 해결방안이 떠오르는것에 존경심이 올라갔다.
- user Service 및 userRepository 테스트시 데이터 베이스에 더미 데이터를 넣어서 작업하는 것 보단 직접적으로 데이터를 작성해서 넣는게 더빠르다는걸 알았다.
